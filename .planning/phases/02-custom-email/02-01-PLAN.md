<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Audit GoDaddy DNS and set up Zoho Mail account with domain verification</name>
  <action>
This task sets up the Zoho Mail account and verifies domain ownership. All steps are in the GoDaddy DNS Manager and Zoho Admin Console (web browsers).

**Step 1 — Audit existing GoDaddy DNS records:**
1. Log in to GoDaddy > My Products > no-bshomes.com > DNS
2. Look for ANY existing MX records — write down their values
3. Look for ANY existing TXT records that start with `v=spf1` — write down their values
4. Look for ANY email forwarding entries (GoDaddy > Email > Forwarding section)
5. DELETE any existing MX records (GoDaddy may have pre-populated defaults)
6. DELETE any existing SPF TXT records (records starting with `v=spf1`)
7. DELETE any email forwarding entries
8. This clears the way for Zoho records without conflicts (see Pitfalls 3 and 4 in research)

**Step 2 — Create Zoho Mail account:**
1. Go to https://www.zoho.com/mail/
2. Click "Sign Up Free" (or "Get Started Free")
3. If free plan is not available in your region, the lowest paid tier (Mail Lite) is ~$1/user/month
4. Sign up using your personal email as the admin account
5. When prompted, enter `no-bshomes.com` as the organization domain

**Step 3 — Verify domain ownership:**
1. Zoho will show a verification TXT record value (e.g., `zoho-verification=zb12345678.zmverify.zoho.com`)
2. Go to GoDaddy DNS Manager
3. Add a new TXT record: Host = `@`, Value = the verification string from Zoho
4. TTL = 1 Hour
5. Wait 5-15 minutes for propagation
6. Return to Zoho Admin Console and click "Verify"
7. Once verified, you may delete the verification TXT record (it is no longer needed)
  </action>
  <verify>
    Zoho Admin Console shows domain `no-bshomes.com` with status "Verified" (green checkmark or similar indicator).
  </verify>
  <done>Zoho Mail account exists with no-bshomes.com verified as the organization domain. GoDaddy DNS is clean of conflicting MX/SPF/forwarding records.</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Add all DNS records at GoDaddy and create mailboxes in Zoho</name>
  <action>
This task adds all required DNS records and creates the three mailboxes. Follow this exact order.

**Step 1 — Add MX records at GoDaddy:**
Add three MX records in GoDaddy DNS Manager:
1. Type: MX, Host: `@`, Points to: `mx.zoho.com`, Priority: `10`
2. Type: MX, Host: `@`, Points to: `mx2.zoho.com`, Priority: `20`
3. Type: MX, Host: `@`, Points to: `mx3.zoho.com`, Priority: `50`

**Step 2 — Add SPF record at GoDaddy:**
IMPORTANT: There must be exactly ONE SPF record. If any `v=spf1` TXT record exists, delete it first.
1. Type: TXT, Host: `@`, Value: `v=spf1 include:zohomail.com -all`
2. Note: Use `include:zohomail.com` (NOT `include:zoho.com` — this is a common mistake)

**Step 3 — Generate and add DKIM record:**
1. In Zoho Admin Console, go to Domains > no-bshomes.com > Email Configuration > DKIM
2. Click "Add" to generate a new DKIM key
3. Choose selector name: `zoho` (or note whatever name you choose — needed for verification later)
4. Choose 2048-bit key if offered
5. Zoho will display the TXT record value (starts with `v=DKIM1; k=rsa; p=...`)
6. Go to GoDaddy DNS Manager
7. Add TXT record: Host: `zoho._domainkey` (use your selector name), Value: the full DKIM value from Zoho
8. IMPORTANT: This is a TXT record, NOT a CNAME
9. Return to Zoho Admin Console and click "Verify" for DKIM (may take 15-60 minutes to propagate)

**Step 4 — Add DMARC record at GoDaddy:**
1. Type: TXT, Host: `_dmarc`, Value: `v=DMARC1; p=none; rua=mailto:contact@no-bshomes.com`
2. Start with `p=none` (monitoring only) — tighten to `p=quarantine` after 30 days of clean reports

**Step 5 — Create mailboxes in Zoho Admin Console:**
1. Go to Zoho Admin Console > Users
2. Create user/mailbox: `contact@no-bshomes.com`
3. Create user/mailbox: `brian@no-bshomes.com`
4. Create user/mailbox: `shawn@no-bshomes.com`
5. Set passwords for each account
6. Confirm all three show as "Active" in the user list

**Step 6 — Wait for DNS propagation:**
DNS changes take 15 minutes to 48 hours to fully propagate. MX records typically resolve within 15-60 minutes. DKIM may take longer.
Use https://dnschecker.org to check propagation status for:
- MX record for `no-bshomes.com`
- TXT record for `no-bshomes.com` (SPF)
- TXT record for `zoho._domainkey.no-bshomes.com` (DKIM)
- TXT record for `_dmarc.no-bshomes.com` (DMARC)
  </action>
  <verify>
    GoDaddy DNS Manager shows: 3 MX records (mx.zoho.com, mx2.zoho.com, mx3.zoho.com), 1 SPF TXT record, 1 DKIM TXT record, 1 DMARC TXT record. Zoho Admin Console shows 3 active mailboxes.
  </verify>
  <done>All 6 DNS records are configured at GoDaddy. Three mailboxes (contact@, brian@, shawn@) are created and active in Zoho. DKIM shows verified in Zoho Admin Console.</done>
</task>
