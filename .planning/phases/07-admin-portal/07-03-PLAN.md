Create `src/app/api/admin/upload-url/route.ts` (POST):
- Import { auth } from "@/auth"
- Check session: `const session = await auth(); if (!session?.user?.email) return new Response("Unauthorized", { status: 401 });`
- Read { filename, contentType } from request JSON
- Generate blobName: `gallery/${Date.now()}-${crypto.randomUUID()}.${ext}`
- Use StorageSharedKeyCredential with AZURE_STORAGE_ACCOUNT_NAME and AZURE_STORAGE_ACCOUNT_KEY
- Generate SAS with BlobSASPermissions.parse("cw"), 15-minute expiry, contentType set
- Return JSON: { sasUrl, permanentUrl, blobName }
- CRITICAL: Include `contentType` in SAS parameters so Azure accepts the upload

Create `src/app/api/admin/gallery/route.ts`:
- GET: auth check, query `db.select().from(nbsGalleryImages).orderBy(asc(nbsGalleryImages.displayOrder))`, return JSON array
- POST: auth check, read { blobUrl, alt, projectName } from body, get current max displayOrder and add 1, insert into nbsGalleryImages, return new record as JSON with 201 status

Create `src/app/api/admin/gallery/[id]/route.ts`:
- PUT: auth check, read { alt, projectName } from body, parse id from params, update record with `db.update(nbsGalleryImages).set({ alt, projectName }).where(eq(nbsGalleryImages.id, id))`, return updated record
- DELETE: auth check, parse id from params, delete with `db.delete(nbsGalleryImages).where(eq(nbsGalleryImages.id, id))`, return 204

Create `src/app/api/admin/gallery/reorder/route.ts`:
- PUT: auth check, read { orderedIds: number[] } from body
- Loop through orderedIds and update displayOrder for each: `db.update(nbsGalleryImages).set({ displayOrder: index }).where(eq(nbsGalleryImages.id, id))`
- Return 200

ALL admin API routes must independently verify session (middleware is not sufficient for API routes). Pattern:
```typescript
const session = await auth();
if (!session?.user?.email) return new Response("Unauthorized", { status: 401 });
```
