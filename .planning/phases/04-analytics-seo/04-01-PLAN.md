1. Update `src/components/ContactForm.tsx`:
   - Import `sendGAEvent` from `@next/third-parties/google`
   - After the successful Netlify submission (after `if (!netlifyRes.ok)` check passes, right before the fire-and-forget HouseFinder fetch), add:
     ```tsx
     sendGAEvent('event', 'generate_lead', {
       event_category: 'contact_form',
       event_label: 'homepage_contact',
       value: 1,
     })
     ```
   - This must be AFTER the Netlify success check but BEFORE `setSubmitted(true)` — so the event only fires on actual successful submissions.

2. Update `src/app/faq/page.tsx`:
   - Add a FAQPage JSON-LD `<script>` tag. Build the schema dynamically from the existing `faqs` array so it stays in sync automatically:
     ```tsx
     const faqSchema = {
       '@context': 'https://schema.org',
       '@type': 'FAQPage',
       mainEntity: faqs.map(faq => ({
         '@type': 'Question',
         name: faq.question,
         acceptedAnswer: {
           '@type': 'Answer',
           text: faq.answer,
         },
       })),
     }
     ```
   - Inject with a `<script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(faqSchema) }} />` inside the JSX (after `<Header />` or before `</>`). Do NOT use `next/script` here since this is a client component — use a plain `<script>` tag which is valid in React 19.
   - Actually, since the FAQ page is a client component ("use client"), and we want the JSON-LD to be in the initial HTML for crawlers: extract the `faqs` array to a separate constant file `src/lib/faq-data.ts` (just move the array, export it). Then the FAQ page imports it. The JSON-LD script tag with `dangerouslySetInnerHTML` will still render server-side on initial page load in Next.js even in a client component because Next.js SSRs client components.
   - Alternatively (simpler): keep `faqs` in the same file. The `<script type="application/ld+json">` with `dangerouslySetInnerHTML` will be server-rendered by Next.js on the initial HTML response, which is what crawlers see. No extraction needed.
   - Go with the simpler approach: keep faqs in-file, add the script tag.
