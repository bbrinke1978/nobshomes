Create the file at .planning/phases/05-voice-lead-pipeline/apps-script/voicemail-to-lead.gs with a complete, production-ready Google Apps Script. The script must:

**Configuration (top of file):**
```javascript
// Configuration — set API_KEY as a Script Property (File > Project settings > Script properties)
// Key: API_KEY  Value: (the HOUSEFINDER_API_KEY from Netlify env vars)
var CONFIG = {
  API_URL: 'https://housefinder-app.azurewebsites.net/api/leads',
  GMAIL_SEARCH: 'from:voice-noreply@google.com label:inbox -label:vm-processed',
  PROCESSED_LABEL: 'vm-processed',
  DEFAULT_STATE: 'UT'
};
```

**Main function: processVoicemails()**
1. Get or create the "vm-processed" Gmail label
2. Search Gmail using CONFIG.GMAIL_SEARCH
3. For each thread, for each message:
   a. Extract caller phone from subject using flexible regex:
      - Match `+1XXXXXXXXXX` format (strip +1 prefix)
      - Match `(XXX) XXX-XXXX` format (strip parens, spaces, dashes)
      - Match any 10+ digit sequence as fallback
      - TODO comment: "Adjust regex after inspecting actual voicemail email format"
   b. Extract transcription from plain body:
      - Get message.getPlainBody()
      - Strip everything after "Play message" or "YOUR ACCOUNT" or known footer markers
      - Trim whitespace
      - TODO comment: "Adjust body parsing after inspecting actual voicemail email"
   c. If no phone number extracted, log warning and skip (don't create junk leads)
   d. Build payload: { phone, message (transcription, max 2000 chars), source: "voicemail" }
   e. POST to API with x-api-key header from Script Properties
   f. Log response status
   g. Apply "vm-processed" label to the thread
   h. If API call fails (non-2xx), log error but still label the email (prevent retry storms). Include the error in the log.

**Helper function: getOrCreateLabel(name)**
Returns GmailApp.getUserLabelByName(name) or creates it if missing.

**Helper function: getApiKey()**
Returns PropertiesService.getScriptProperties().getProperty('API_KEY'). Throws descriptive error if not set.

**Helper function: extractPhoneNumber(subject)**
Extracts and normalizes phone number from subject line. Returns 10-digit string or null.

**Helper function: extractTranscription(body)**
Extracts transcription text from email body. Returns trimmed string.

**Error handling:**
- Wrap entire processVoicemails in try/catch, log errors with console.error
- Individual message processing wrapped in its own try/catch so one bad email doesn't stop the batch
- If API_KEY is not set, log clear error and return early
